Key Decisions Made

Decision 1: Single Shared Context MCP

ONE Context MCP server with namespace isolation, NOT 20 separate instances. 10× cheaper at scale.

Decision 2: HTTP API Gateway for iOS

Deploy at context-api.arcfoundry.ai for iOS/external access. MCP is server-to-server only.

Decision 3: ADC Gateway as Deployment Gate

All agents must be registered in PostgreSQL ADC registry before accessing Context MCP.

Decision 4: Platform Covenants

4 inviolable rules govern platform behavior: Data≠Context, Domain Isolation, Immutable Versions, Audit Everything.

4 Inviolable Platform Rules

These covenants govern all platform behavior and cannot be overridden by any product or agent.

Covenant 1: Data ≠ Context

Rule: Product data (interview responses, user state) NEVER enters Context MCP storage.

Enforcement: ADC Gateway blocks unauthorized writes. Violations logged to audit trail.

Violation Response: BLOCK + ESCALATE

Covenant 2: Domain Isolation

Rule: Agents can only access domains explicitly permitted in their ADC.

Enforcement: Per-agent domain permissions checked on every request.

Violation Response: BLOCK + LOG

Covenant 3: Immutable Methodology Versions

Rule: Published methodology versions (cold tier) are immutable. Updates create new versions.

Enforcement: 409 Conflict on overwrite attempts.

Violation Response: BLOCK

Covenant 4: Audit Everything

Rule: All Context MCP operations logged to PostgreSQL with agent, domain, operation, timestamp.

Enforcement: Middleware logs before routing. Missing audit = request fails.

Violation Response: AUTO-ENFORCE

Violation Response Matrix

Response Action Notification
WARN Log violation, allow request Weekly digest
BLOCK Reject request with 403/409 Immediate to agent owner
ESCALATE Block + alert platform team Immediate to platform admins
QUARANTINE Disable agent until manual review Immediate + incident ticket
